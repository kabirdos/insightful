Context

The live homepage is clear and lightweight, but it mostly behaves like a browse feed with an upgrade panel attached underneath. The richer `insight-harness` data gives us enough signal to sell distinct developer identities: tool mix, skills, plugins, models, subagent behavior, prompt shape, file operation style, and runtime posture. These concepts explore four different ways to make that visible.

Four homepage directions for turning Insight Harness from a profile list into a recognizable developer-signal product.

Each option changes both the look and the information model. The goal is not "prettier cards." The goal is to decide what the home page is fundamentally for: ranking operators, telling stories, mapping patterns, or converting visitors into participants.

Live Homepage Read

Option A

Signal Board

Display logic: lead with category slices and benchmark-style profile modules, not generic cards.

Option B

Feel: reflective, credible, human, learnable.
Display logic: lead with stories, signatures, and "what this harness says about the person."

What It Optimizes For

Option C

Pattern Atlas. Treat the homepage as a map of how people use AI coding tools, not just a list of people. The emotional promise is: "I can explore the space of workflows." This is the most data-native option.

Pattern Atlas

Look: bright systems-map UI with category tiles and pattern bands.
Feel: exploratory, analytical, modern, map-like.
Display logic: visitors discover by workflow pattern first, individual profile second.

What It Optimizes For

Teaching the shape of the ecosystem, not just surfacing examples.

What Changes

Option D

Build Then Browse. Treat the home page as a conversion engine for creating better public harnesses. The emotional promise is: "I can understand the product, generate my own profile, and then explore others." This is the most productized and funnel-aware direction.
