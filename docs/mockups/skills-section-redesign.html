# skill-showcase

Build a single-file public HTML showcase of a collection of Claude Code skills — table of contents, grouped categories, per-skill README with hero image, custom-vs-plugin labeling, and PII scrubbing.

## Overview This skill takes a directory of Claude Code skills and produces a portable HTML file suitable for sharing. Includes: table of contents, grouped categories, per-skill README rendering with inlined hero image, custom-vs-plugin source labeling, and a privacy pass that scrubs identifiers, paths, and other PII from skill descriptions and README content.

## Usage ```bash skill-showcase --skills-dir ~/.claude/skills --output ~/showcase.html ```

## Options - `--skills-dir` — path to directory containing skill folders - `--output` — output file path (default: ./showcase.html) - `--title` — page title (default: "My Claude Skills") - `--include-categories` — comma-separated list of categories to include - `--exclude-private` — skip skills marked as private...

skill-showcase

Build a single-file public HTML showcase of a collection of Claude Code skills — table of contents, grouped categories, per-skill README with hero image, custom-vs-plugin labeling, and PII scrubbing.

Overview

Takes a directory of skills and produces one portable HTML file you can host anywhere. No build step. No runtime dependencies. Designed for sharing your Claude Code setup publicly without leaking identifiers.

Quick start

skill-showcase \
  --skills-dir ~/.claude/skills \
  --output ~/showcase.html

Flags

Flag Default Purpose
--skills-dir ~/.claude/skills Directory containing skill folders
--output ./showcase.html Output HTML path
--title "My Claude Skills" Page title
--exclude-private false Skip skills marked private

Privacy

Ships with a built-in scrubber that masks filesystem paths, email addresses, API keys, and usernames before rendering. Safe to publish.

What gets scrubbed

  • Absolute paths under /Users/*, /home/*
  • Email addresses and phone numbers
  • Anything matching known API-key prefixes
  • Git remote URLs with credentials
Ingredients: Inter font · h1 22px / h2 17px with rule / h3 15px · inline code pink-on-slate · fenced pre dark slate · styled tables and blockquotes · hero image at 10px radius. All already safe via existing rehype-sanitize schema — this is CSS-only.
